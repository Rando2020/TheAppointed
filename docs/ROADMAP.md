# ProjectTactic Roadmap

## Phase 0: Tactical foundation
Goal: create the tactical architecture without breaking the existing prototype.

Deliverables:
- Data-driven maps.
- Terrain definitions.
- Unit/job/ability/enemy data.
- Grid utilities.
- Pathfinding utilities.
- Turn order utilities.
- Battle shell components.
- Debug-friendly vertical slice docs.

## Phase 1: First playable tactical battle
Goal: one complete grid battle from story to victory screen.

Deliverables:
- 10x8 tactical map.
- Two player units.
- Two enemies.
- Movement range.
- Basic attack.
- Ability targeting.
- Damage preview.
- Turn timeline.
- Victory condition.
- Rewards.

## Phase 2: Modern UX pass
Goal: make the prototype feel modern and readable.

Deliverables:
- Enemy intent preview.
- Undo movement.
- Battle speed controls.
- Tile coordinate debug overlay.
- Status tooltip panel.
- Accessibility settings.
- Save/load.

## Phase 3: Elemental terrain vertical slice
Goal: make Vaelthar feel distinct.

Deliverables:
- Wet terrain.
- Frozen terrain.
- Electrified terrain.
- Burning terrain.
- Elemental reaction preview.
- Mirefen marsh mission.

## Phase 4: Job and progression layer
Goal: add long-term tactics RPG depth.

Deliverables:
- JP gain.
- Job levels.
- Ability unlocks.
- Job board.
- Equipment panel.
- Party roster.

## Phase 5: Story/town/world integration
Goal: connect narrative and progression.

Deliverables:
- Ashvale loop.
- Mission board.
- Quest completion.
- Town unlocks.
- Codex and glossary.
- Faction reputation placeholders.

## Phase 6: Visual identity pass
Goal: replace placeholders with consistent original art direction.

Deliverables:
- Tileset.
- Unit sprites or tokens.
- Portraits.
- UI skin.
- VFX pass.
- Audio pass.
