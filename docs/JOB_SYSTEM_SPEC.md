# Job System Specification

## Intent
The job system should reward experimentation without overwhelming the player. Jobs define battlefield role, stat growth, ability access, reaction options, passives, equipment permissions, and ascension paths.

## Job structure
Each job should include:
- ID
- Display name
- Tier
- Role
- Movement and jump modifiers
- Primary stats
- Equipment permissions
- Ability list
- Reaction ability
- Passive ability
- JP costs
- Unlock requirements
- Ascended job path

## Base job principles
Base jobs should be simple to understand and valuable through the full game.

Initial base jobs:
- Warder: physical tank, guard, sunder, cover.
- Arcanist: elemental caster, Ether pressure, combo setup.
- Resonant: Guardian bond, anchor shatter, summon access.
- Luminary: healing, Ether restoration, cleanse, revive.
- Skywarden: jump attacks, anti-backline, mobility.
- Chronist: timeline control, haste, slow, delay.
- Oathbound: aura support, counterplay, formation bonuses.
- Voidcaller: curse, drain, risk-reward dark magic.
- Null Resonant: forbidden hybrid, anti-Guardian tools.

## Ascended job principles
Ascended jobs should require mastery of multiple related jobs. They should not simply be stronger versions. They should unlock new tactical patterns.

Example unlocks:
- Null Breaker: Warder level 4 + Voidcaller level 2.
- Etherweaver: Arcanist level 4 + Chronist level 3.
- Primal Binder: Resonant level 5 + Luminary level 3.
- Seraph: Luminary level 5 + Oathbound level 3.

## JP rules
- JP is earned per battle, per action, and through objectives.
- JP can be job-specific and/or shared as overflow.
- Story battles can award bonus JP for optional objectives.

## Modern UX
Players should be able to preview:
- Unlock tree.
- JP needed for next ability.
- What a job contributes tactically.
- Which maps or enemies favor that job.
